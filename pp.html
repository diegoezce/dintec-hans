PROPUESTA DE PRODUCTO

Robot Companion

Puente digital entre la documentación de ingeniería y el operador en piso. Centraliza información de mantenimiento, auditoría y soporte para flota multi-marca de robots industriales.

```

La Visión

Los integradores que venden múltiples marcas de robots enfrentan un problema que nadie resuelve: después de la venta, el operador en piso no tiene acceso simple a documentación, procedimientos de troubleshooting, ni forma rápida de reportar problemas.

Robot Companion conecta el MOC (sistema de mantenimiento existente) con el operador mediante un acceso simple, rápido y multi-marca.

El Problema

Hoy: El operador se queda sin herramientas

Cuando aparece una alarma a las 3 AM:

  • ¿Busca en carpetas compartidas?
  • ¿Llama al integrador?
  • ¿Espera al ingeniero?
  • ¿Improvisa?

El conocimiento existe, pero está en silos: documentación del fabricante, notas del integrador, manual de seguridad, historial de mantenimiento—todo desconectado.

Los dolores específicos

Integrador multi-marca: 15 robots de 5 fabricantes diferentes. Sin forma unificada de acceder a documentación.

Auditoría/Compliance: ¿Cuándo se hizo el último mantenimiento? ¿Quién? ¿Qué se encontró?

Soporte: Operador no reporta problemas porque no hay canal claro.

Solución: Robot Companion

¿Cómo funciona?

1. Etiqueta en cada robot

Cada robot tiene un QR único (o RFID en futuro). Impreso, adhesivo, durable. Costo: $0.50-2 por unidad.

2. App móvil

Operador escanea → accede a toda la información. Funciona con WiFi o sin conexión (modo offline).

3. Panel web

Integrador y operador ven historial completo, auditorías, work orders, documentación centralizada.

4. Integración con MOC

Se conecta con el sistema de mantenimiento existente. No reemplaza, complementa.

Qué puede hacer el operador

  • ✓ Ver documentación (manual, parámetros, límites de seguridad)
  • ✓ Ver alarmas activas e historial de errores
  • ✓ Hacer checklist de pre-arranque
  • ✓ Registrar auditorías (fotos, notas, firma)
  • ✓ Reportar problemas (tienda en app, envía a integrador)
  • ✓ Ver últimas acciones de mantenimiento
  • ✓ (Futuro) Control remoto básico del robot
  • ✓ (Futuro) Copilot IA para troubleshooting

Propuestas & Alternativas

Cuatro formas de llevar esta idea al mercado, cada una con trade-offs distintos:

Opción A QR + App Standalone (MVP Puro)

App móvil simple. Escanea QR → accede a documentación estática y formularios. No integración con MOC.

✓ Ventajas
  • Rápido de desarrollar (4-8 semanas)
  • Costo bajo ($30-50K desarrollo inicial)
  • Fácil de adoptar (solo smartphone)
  • No requiere integración compleja
  • El integrador controla el contenido
✗ Limitaciones
  • No conecta con historial de mantenimiento existente
  • Datos duplicados (copia manual de documentación)
  • No es fuente única de verdad
  • Sin analítica de uso/tendencias

Ideal para: Piloto rápido. Probar con integrador pequeño (5-10 robots) antes de comprometerse.

Opción B QR + App + API Integration (Recomendado)

App como Opción A, pero con APIs que se conectan al MOC/CMMS del cliente (Oxmaint, SAP, etc.).

✓ Ventajas
  • Conecta con realidad operativa (historial vivo)
  • Fuente única de verdad
  • Escalable a múltiples integradores
  • Datos para analítica/predictivo
  • Modelo de suscripción viable
✗ Limitaciones
  • Desarrollo más largo (12-16 semanas)
  • Costo inicial mayor ($80-120K)
  • Requiere integración API (cada cliente puede ser diferente)
  • Soporte post-venta más complejo

Ideal para: Scalability. Modelo B2B viable. Precio: $200-500/mes por integrador.

Opción C RFID + Control Remoto (Premium)

Evolución. RFID en lugar de QR (lectura automática sin línea de vista). Incluye capacidad de control remoto básico del robot.

✓ Ventajas
  • Diferenciador claro vs competencia
  • Datos más ricos (automáticos, no manuales)
  • Caso de uso remoto (soporte desde lejos)
  • Premium pricing justificado
✗ Limitaciones
  • Desarrollo técnico más complejo (control de robots)
  • Hardware RFID (lectores, etiquetas, instalación)
  • Costo total: $300-500K+ (infraestructura)
  • Ciclo de venta mucho más largo
  • Complejidad de seguridad (control remoto = riesgo)

Ideal para: Fase 2 o 3. Después de validar Opción B.

Opción D White-Label para OEM (Largo plazo)

Vender la plataforma como servicio a los fabricantes de robots (FANUC, ABB, KUKA). Ellos lo branded para sus clientes.

✓ Ventajas
  • Escala masiva (miles de robots)
  • Ingresos recurrentes alto volumen
  • Integración nativa con los robots
  • Posición estratégica en la cadena
✗ Limitaciones
  • Acceso a OEMs muy difícil sin traction primero
  • Ciclos de venta de 12-24 meses
  • Dependencia de OEM para adopción
  • Requiere muy buen producto ya validado

Ideal para: Objetivo de 3-5 años. No es salida inicial.

Recomendación

Camino de 3 fases:

1

Fase 1: MVP (Opción A) - Piloto rápido

Duración: 3 meses | Inversión: $40K

Con tu cuñado: Implementar en su operación. 15 robots, QR + app. Validar que el dolor es real y que la solución lo resuelve.

Resultado: Case study. Evidencia de que funciona. Datos de uso real.

2

Fase 2: Escalabilidad (Opción B) - Producto productivo

Duración: 4 meses después | Inversión: $100K adicionales

Con integradores: Ir a mercado con API. Vender a otros integradores. Modelo SaaS ($300/mes por operador o $500/mes por integrador).

Resultado: 5-10 integradores piloto. Ingresos mensuales. Validación B2B.

3

Fase 3: Expansión (Opción C o D) - Premium

Duración: 6-12 meses después | Decisión: RFID o White-label

Basado en: Feedback de Fase 2. Si piden control remoto → Opción C. Si OEMs preguntan → Opción D.

Resultado: Escalada seria a mercado masivo o partnership estratégico.

Próximos Pasos Concretos

Esta semana:

  • Hablar con tu cuñado: "¿Cuándo un operador necesita información, qué le falta hoy?"
  • Mapear sus 15 robots: marcas, antigüedad, dónde están las grietas
  • Ver su sistema actual de mantenimiento (¿usa CMMS?, ¿Excel?, ¿nada?)
  • Estimar: si implementamos esto, ¿cuánto downtime evitamos? ¿Cuánto cuesta?

En 2 semanas:

  • Prototipo visual (wireframes/mockup de la app)
  • Decidir: ¿A dónde va primero? MVP simple o API desde el inicio
  • Tech stack: qué tecnología. Recomendación: React (app) + Node.js (backend) + PostgreSQL

En 1 mes:

  • Primer sprint de desarrollo (Opción A)
  • QR generados e impresos para prueba
  • App básica funcional (escanear → datos)

Mercado

Dolor real

Tecnología existente

Ventaja competitiva
